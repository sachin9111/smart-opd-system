# WHERE EUREKA COMES

Every service starts.

↓

↓

Eureka

Appointment Service


↓

Registers


Eureka


API Gateway never needs fixed URLs.

Instead of


http://localhost:8082


it asks Eureka:


Where is USER-SERVICE?


Eureka replies


localhost:8082


Later, if you have two instances:


USER-SERVICE

8082

8085


Gateway automatically load-balances between them.

# WHERE KAFKA COMES

Instead of


Auth Service

↓

calls

↓

User Service


we do


Auth Service

↓

Kafka

↓

User Service


Benefits:

- Services are loosely coupled.
- If User Service is temporarily down, Kafka keeps the event until it can be processed.
- Multiple services can react to the same event independently.

This is the architecture we'll build step by step. Right now, you already have Eureka, Kafka, Auth Service, and User Service started. The next milestone is to complete the Kafka-based communication between Auth Service and User Service so that a new user registration automatically creates the corresponding patient profile without any direct REST call. After that, we'll add the API Gateway and continue expanding the remaining services.
